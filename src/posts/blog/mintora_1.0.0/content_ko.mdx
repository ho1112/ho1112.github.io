<h1>왜 만들었는가</h1>

개발 블로그를 개설했습니다.
평소 개발자로서 일하면서 학습한 내용을 체계적으로 정리하고 기록할 공간이 필요하다고 느껴왔습니다.
개인적으로 공부한 내용을 더 잘 기억하고 싶었고 개발자로서의 성장을 눈에 보이는 형태로 쌓아가고 싶었습니다.
또 앞으로의 커리어에 도움이 될 수 있도록 포트폴리오 용도로도 활용되면 좋다고 생각했습니다.
그리고 이러한 여러 목적과 이유로 고민 끝에 선택한 방식은 GitHub Pages + Next.js + MDX입니다. 이번 글에서는 제가 이 방법을 선택한 이유와 사용한 기술들에 대해서 기록하려고 합니다.

## github page
블로그 플랫폼을 고를 때 여러 가지 옵션이 있었습니다.
티스토리, 네이버 블로그 같은 전통적인 플랫폼부터 미디엄(Medium), velog와 같은 개발자가 많이 사용하는 플랫폼 또 노션(Notion), 옵시디언(Obsidian)와 같이 블로그 기능을 지원하는 메모앱
일본이라면 hatena와 note가 있습니다.
그리고 배포할 곳도 aws, vercel까지 다양한 방법이 있었지만 저는 GitHub Pages를 선택했습니다. 그 이유는 다음과 같습니다.

### 커스터 마이징
gitHub Pages는 HTML, CSS, JavaScript 기반의 정적 웹사이트를 호스팅할 수 있는 무료 서비스입니다. 일반적인 블로그 서비스들과 다르게 내 마음대로 스타일과 기능을 커스터마이징할 수 있습니다. 프론트엔드 개발자로서 이 자유도는 큰 장점이라고 생각했습니다.
그리고 블로그 자체를 직접 만들고 자유롭게 만든 기능, 페이지, 컴포넌트 등도 업로드 하기에 최적화 됐다고 생각했습니다.

### 나만의 개발기록
기존 블로그 플랫폼들은 서비스가 언제 중단될지 모른다는 리스크가 있습니다. 반면 gitHub는 전 세계 개발자들이 사용하는 플랫폼이므로 서비스가 갑자기 종료될 확률이 적다고 생각했습니다.
그리고 소스코드 기록까지 전부 확인할 수 있다는 점이 큰 메리트였습니다.

### 다국어 라우팅
한국어와 일본어 두 가지 언어로 포스트를 작성할 생각인데 기존의 플랫폼으로 만들면 어느 한 쪽으로 분리하기 어려웠습니다.
라우팅만으로 언어를 독립적으로 유지하기 위해선 github page가 최선이라고 생각합니다.

## 기술스택을 결정한 이유
### Next.js
Next.js는 현재 제가 현업에서 사용하는 주력 기술입니다. 정적 사이트 생성(SSG) 기능을 제공하기 때문에 블로그 개발에도 좋고 현업과 연계되기 떄문에 처음부터 이 기술을 베이스로 만들어야겠다고 생각했습니다.

### mdx
mdx는 Markdown을 확장한 형식으로 마크다운 문서 내에서 JSX 코드를 직접 사용할 수 있게 합니다. 덕분에 글을 작성할 때 코드 예시나 커스텀 컴포넌트를 쉽게 넣을 수 있어 블로그 작성에 최적화된 환경을 제공합니다. 특히 개발 포스트를 작성할 때 코드 하이라이팅이나 예시 컴포넌트를 쉽게 넣을 수 있어 편했고 일관된 작성법을 사용하는 것이 통일감을 주기 때문에 좋다고 생각했습니다.

### tailwind CSS
tailwind CSS는 최근 프론트엔드 개발에서 각광받는 CSS 프레임워크입니다. 예전에 현업에서 한 번 사용해본 경험이 좋았고 언젠가 실무에서도 사용할법한 기술이라 경험을 쌓아두고 싶었습니다. 간편한 클래스 기반의 유틸리티 CSS 방식이고 css파일을 따로 작성하지 않아도 되기 때문에 빠르게 스타일링을 완료할 수 있었습니다.

## 블로그 기능
### 필수기능
- 카테고리
  카테고리는 우선 5개를 만들었습니다.
  HOME: TOP페이지, 모든 포스터를 최신순으로 표시합니다.
  weekly: 주말에 IT기술동향과 같은 포스트를 작성해보려고 만들었습니다.
  webDev
    workLog: 현업에서 일하면서 학습한 내용을 정리해둘 곳입니다.
    realeseNote: nextjs, react 등 사용하는 기술들의 릴리스 내용을 번역, 정리
    FE: 프론트엔드에 도움이 될만한 포스트
    DevOps: aws, docker 등의 포스트
    techStory: 평소에 궁금했던 점, 개발 정보 등(weekly와 겹치는 부분이 있어서 둘 중에 하나는 없앨 수도 있습니다)
  blog: 블로그 업데이트
  codeLap: 만들어 보고 싶은, 해보면 좋을 기능들을 가볍게 만들어 올리는 곳
- TOC(table of content)
  요즘 블로그는 이 기능이 필수인 것 같습니다. 포스트의 내용을 한 눈에 살피기도 좋고 이동하기에도 좋아 꼭 필요하다고 생각합니다.
- 댓글
  개인 기록용으로 만들었기 때문에 필수적이지는 않지만 블로그이니만큼 댓글 기능은 만들어두고 싶었습니다.
- 조회수 확인
  hits.sh와 같은 서비스로 간단히 구현가능 했기에 추가했습니다.

### 추가하고 싶은 기능
- 검색기능(제목+내용)
  개인 블로그 중에 타이틀로 검색하는 곳은 일부 있었지만 내용까지 검색가능한 곳은 상당히 드문 것 같습니다.
  정보도 찾기 힘들어보였지만 불가능하진 않아보였기에 포스트 수가 많아지면 꼭 추가하고 싶습니다.
- 날씨 연동
  헤더 부분에 날씨를 표시해서 그에 맞게 이펙트를 넣어보고 싶은데 사용자의 위치 정보를 기반으로 하면 권한요청 문제가 생기기 때문에 도쿄 고정으로 해볼까 생각 중입니다.
- 스크롤 상승 아이콘
- 랜덤 버튼
