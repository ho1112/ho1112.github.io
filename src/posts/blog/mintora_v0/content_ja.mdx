<h1>なぜ作ったのか</h1>

github pageでブログを開設しました。
普段開発者として働きながら学習した内容を体系的に整理して記録する空間が必要だと感じてきました。
個人的に勉強した内容をもっとよく覚えたかったし、開発者としての成長を目に見える形で築いていきたいと思いました。
また、これからのキャリアに役立つように、ポートフォリオの用途としても活用されれば良いと思いました。
そして、このような様々な目的と理由で悩んだ結果、選択した方法はGitHub Pages + Next.js + MDXです。今回は私がこの方法を選択した理由と使用した技術について記録しようと思います。

---
## github page
ブログ プラットフォームを選ぶとき、いくつかのオプションがありました。
Tistory, Naverのような伝統的なプラットフォームからミディアム(Medium)、 velogのような開発者が多く使用するプラットフォーム、 またNotion, Obsidianのようなブログ機能をサポートするメモアプリ
日本ならhatenaとnoteがありました。
そしてデプロイするところもaws, vercelまで様々な方法がありましたが、私はGitHub Pagesに決めました。 その理由は次の通りです。

### カスタマイズ
gitHub Pages는はHTML, CSS, JavaScriptベースの静的ウェブサイトをホスティングできる無料サービスです。一般的なブログサービスとは異なり、自分の好きなスタイルと機能をカスタマイズすることができます。フロントエンドの開発者として、この自由度は大きな長所だと思いました。
そしてブログ自体を自分で作って自由に作った機能、ページ、コンポーネントなどもアップロードするのに最適化されたと思いました。

### 自分だけの開発記録
既存のブログプラットフォームは、サービスがいつ中断されるかわからないというリスクがあります。 一方、gitHubは世界中の開発者が使用するプラットフォームなので、サービスが突然終了する確率が少ないと思いました。
そしてソースコードの記録まで全部確認できるという点が大きなメリットでした

### 多言語ルーティング
日本語と韓国語、2つの言語でポストを作成するつもりですが、既存のプラットフォームにするとどちらか一方に分離するのが難しかったです。
ルーティングだけで言語を独立的に維持するためには、github pageが最善だと思いました。

---
## スキルスタックを決定した理由
### Next.js
Next.jsは現在私が現場で使用している主要技術です。静的サイト生成(SSG)機能を提供するのでブログ開発にもよく、仕事とも連携されるので最初からこのスキルをベースに作らなければならないと思いました。

### mdx
mdxはMarkdownを拡張した形式でマークダウン文書内でJSXコードを直接使用できるようにします。 特に開発ポストを作成するとき、コードハイライトや例示コンポーネントを簡単に入れることができて楽でしたし、一貫した書き方を使うことが統一感を与えるので良いと思いました。

### tailwind CSS
tailwind CSSは最近フロントエンド開発で脚光を浴びているCSSフレームワークです。以前、現場で一度使ってみた経験が良かったし、いつか実務でも使えそうな技術なので、経験を積んでおきたいと思いました。簡単なクラスベースのユーティリティCSS方式であり、cssファイルを別途作成しなくても済むため、早くスタイリングを完了することができました。

## 構造
![](/posts/blog/mintora_v0/directory.avif)
- public
  イメージファイルを管理します。posts下のファイルは実際ルーティングとマッピングされます。
- src
  - app
  全般的なレイアウトを担当(layout.tsx, not-found.tsx など)
  - components
  共通コンポーネント、mdx、リスト、詳細ページコンポーネント
  - posts
  mdxファイル、カテゴリ/ルーティングにマッピングして使います。

## 機能
### 必須機能
- カテゴリ
  カテゴリはまず5個を作りました。
  HOME: TOPページ、全てのポストを新着順で表示します。
  weekly: 週末にIT技術動向などをまとめる予定で作りました。
  webDev
    workLog: 仕事で学んだ内容をまとめるところです。
    realeseNote: nextjs, reactなどの技術のリリースを翻訳、整理
    FE: フロントエンド関連の技術
    DevOps: aws, dockerなど
    techStory: 気になったトピックや技術ネタ(weeklyと重複するなら消すのも検討)
  blog: ブログアップデート
  codeLap: 作ってみたい、チャレンジしてみたいコード
- TOC(table of content)
  最近のブログはこの機能は必須みたいです。ポストの内容を一目で把握できますし、移動にもいいので、ぜひ必要だと思います。
- コメント
  個人記録用に作ったので必須ではありませんが、一応ブログだからコメント機能は作っておきたかったです。
- 閲覧数表示
  hits.shのようなサービスで簡単に実装できたので、追加しました。

### 追加したい機能
- 検索機能(タイトル+本文)
  個人ブログの中で、タイトルで検索するところは一部ありましたが、本文まで検索できるのはかなり珍しいようです。
  情報も少なかったですが、不可能ではないと思うので、ポスト数が多くなったらぜひ追加したいです。
- 天気連動
  ヘッダーに天気情報を表示し、それに合わせたエフェクトを入れてみたいですが。ユーザーの位置情報をベースにすると、権限の問題が発生するため東京固定にしようかと思っています。
- スクロールアップアイコン
- ランダムボタン
