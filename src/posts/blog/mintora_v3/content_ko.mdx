<h1>404 페이지 만들기</h1>

개인적으로 개발을 하며 가장 즐거운 순간 중 하나는 작은 디테일을 나만의 스타일로 꾸며볼 때입니다.
특히 사용자들이 가끔 마주치게 되는 404 에러 페이지도 재밌게 표현해보고 싶었습니다. 이번 포스트에서는 Next.js로 만든 블로그에서 404 페이지에 귀여운 이모티콘을 랜덤하게 표시하는 간단하면서도 재미있는 방법을 소개합니다.

---

## 404 not-found
보통 404 페이지는 사용자가 찾는 페이지가 없을 때 표시됩니다. 많은 사이트들이 기본적으로 단순한 메시지만 띄워주는 데 그치곤 합니다. 하지만 저는 방문자들이 조금이나마 즐겁게 머무르기를 바라는 마음에서 404 페이지도 하나의 디자인 요소로써 신경 쓰고 싶었습니다.
그래서 Google Fonts의 404 페이지를 참고하여 랜덤으로 귀여운 표정을 가진 이모티콘을 띄우도록 구현했습니다.

---
## 코드 예시
```javascript
'use client'

import Link from 'next/link'
import { useEffect, useState } from 'react'

export default function NotFound() {
  const messages = ['｡°(°.◜ᯅ◝°)°｡', '(˃̣̣̣̣︿˂̣̣̣̣ )', '(๑•́ ᎔ ก̀๑)', '╥﹏╥', '•᷄⌓•᷅']
  const [randomMessage, setRandomMessage] = useState('')

  useEffect(() => {
    const message = messages[Math.floor(Math.random() * messages.length)]
    setRandomMessage(message)
  }, [])

  return (
    <div className="flex flex-col items-center justify-center mt-28 text-center px-4">
      <p className="text-[15vw] lg:text-[150px] whitespace-nowrap mb-6">
        {randomMessage}
      </p>
      <h2 className="text-2xl font-semibold mb-4">Not Found</h2>
      <Link
        href="/"
        className="border border-[#e4e4e4] rounded-md text-gray-800 leading-[38px] px-4"
      >
        Home
      </Link>
    </div>
  )
}
```

먼저 준비한 몇 가지 이모티콘을 배열(messages)에 넣습니다. 네이버 스마트보드의 이모티콘 중에서 슬픈 느낌을 가진 것을 골랐습니다.
Math.random() 메서드를 이용해 매번 새로고침할 때마다 배열에서 랜덤한 이모티콘을 선택해 화면에 출력합니다.
이 방식의 가장 큰 장점은 간단한 코드 몇 줄만으로 사용자에게 매번 다른 경험을 줄 수 있다는 것입니다. 사소하지만 이런 작은 디테일은 사이트의 전체적인 인상에 긍정적인 영향을 미친다고 생각합니다.

## 마치며
인터넷이 연결되지 않았을 때 나오는 구글의 공룡 게임과 같이 예상치 못한 상황에서 잠깐이라도 재미를 느낄 수 있다면 더 좋은 사용자 경험(UX)을 제공받게 됩니다.
웹 개발자는 게임처럼 이스터 에그를 넣지는 못하지만 이런 사소한 디테일에서 재미를 느낄 수 있다고 생각합니다.
이런 작은 디테일들이 모인 블로그를 만들고 싶습니다.
