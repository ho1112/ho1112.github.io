<h1>404ページを作成</h1>

個人的に開発で最も楽しい瞬間のひとつは、小さなディテールを自分なりの好みで演出するときです。
特にユーザーがたまに遭遇する404エラーページも、遊び心を加えてみたいと思いました。今回は Next.jsで作成したブログに404ページにかわいい絵文字をランダム表示する、シンプルかつ楽しい方法をご紹介します。

---

## 404 not-found
普段404ページはユーザーが探しているページが存在しない場合に表示されます。多くのサイトが基本的に単純なメッセージを表示するのにとどめたりします。しかし、私は訪問者が少しでも楽しく滞在することを願う気持ちから、404ページも一つのデザイン要素として気を使いたかったのです。
それでGoogle Fontsの404ページを参考してランダムにかわいい表情の絵文字を表示するように実装しました。

---
## コード例
```javascript
'use client'

import Link from 'next/link'
import { useEffect, useState } from 'react'

export default function NotFound() {
  const messages = ['｡°(°.◜ᯅ◝°)°｡', '(˃̣̣̣̣︿˂̣̣̣̣ )', '(๑•́ ᎔ ก̀๑)', '╥﹏╥', '•᷄⌓•᷅']
  const [randomMessage, setRandomMessage] = useState('')

  useEffect(() => {
    const message = messages[Math.floor(Math.random() * messages.length)]
    setRandomMessage(message)
  }, [])

  return (
    <div className="flex flex-col items-center justify-center mt-28 text-center px-4">
      <p className="text-[15vw] lg:text-[150px] whitespace-nowrap mb-6">
        {randomMessage}
      </p>
      <h2 className="text-2xl font-semibold mb-4">Not Found</h2>
      <Link
        href="/"
        className="border border-[#e4e4e4] rounded-md text-gray-800 leading-[38px] px-4"
      >
        Home
      </Link>
    </div>
  )
}

```

まず、用意したいくつかの絵文字を配列(messages)に入れます。NAVERスマートボードの絵文字の中から悲しい感じのものを選びました。
Math.random()メソッドを使って、毎回リフレッシュする度に、配列からランダムな絵文字を選択して画面に表示します。
この方式のメリットはわずかなコード数行だけでユーザーに毎回異なる経験を与えることができるということです。 些細なことですが、このような細かいディテールはサイトの全体的な印象に肯定的な影響を与えると思います。

## おわりに
ネットが繋がらないときに表示されるGoogleの恐竜ゲームのように、予想できなかった状況で少しでも面白さを感じることができれば、より良いユーザー経験(UX)を提供されることになります。
ウェブ開発者はゲームのようにイースターエッグを入れることはできませんが、このような些細なディテールで楽しさを感じることができると思います。
んな小さなディテールが集まったブログを作りたいです。
