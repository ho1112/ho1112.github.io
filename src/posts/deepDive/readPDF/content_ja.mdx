## GeminiはPDFの計算ができない？ファクトチェックと探求の記録

最近、開発者コミュニティを見ていると、AIに関する興味深い話がたくさんあります。その中で私の目を引いたのが、「GeminiはPDFを画像のように丸ごと認識するため数字の計算に弱く、GPTはテキストをそのまま読み込むため計算が得意だ」という噂でした。

エンジニアとしてこのような話を聞くと、「本当にそうなのだろうか？」「技術的に、なぜそのような違いが生まれるのだろうか？」という疑問が湧いてきます。そこで、自ら深く掘り下げてみることにしました。この記事は、私が調査した事実と、それに基づいた私の考えをまとめた探求の記録です。

## 実験設計：両AIを試す「テストシート」

まず、この疑問を明確に解決するために、両AIの能力をきちんと試すことができる特別なテストドキュメントを自作してみました。セルに計算エラーやタイポ、さらには外国語まで盛り込み、複合的な理解能力をテストするように設計しました。

このテストシートを持って、2つのAIに難しい質問を投げかける前に、2つのAIのアプローチを調べてみました。

### ファクト1：両AIはドキュメントを全く異なる方法で見ている

最初に確認したのは、「どのように読み込むか」でした。結論から言うと、両AIがドキュメントを認識する方法は根本的に異なっていました。

#### Geminiのアプローチ：「ネイティブマルチモーダル」

Googleの公式ドキュメントによると、Geminiは **ネイティブマルチモーダル（Native Multimodal）** モデルです。これは、APIがPDFファイルを画像やテキストに変換するプロセスを経ずに、ファイル（`application/pdf`）そのものを直接入力として受け取り処理することを意味します。

この事実が意味するところは明確です。Geminiは、最初からドキュメントを「一つの視覚的情報」として捉えるように設計されているのです。テキスト、画像、表、レイアウトを分離せず、人間が目でページを眺めるように丸ごと理解しようとするアプローチです。これがGeminiの核となるアイデンティティだと私は考えています。

#### GPTのアプローチ：「分析と検証」

GPTの方法は、より分析的です。最新のOpenAI APIはPDFファイルの直接アップロードをサポートしていますが、GPTの強み、特に正確な計算が必要な際の強みは、**コード実行環境（Advanced Data Analysis）** のような外部ツールを活用するところにあります。

これは、GPTの哲学が「正確な分析と検証」に重きを置いていることを示しています。ドキュメントを構成要素として把握し、その中で数値データのように精密さが要求される部分は、コードという最も確実な論理的ツールを通じて処理する方式です。信頼性を高めるための、非常に実用的な選択だと言えるでしょう。

### ファクト2：各アプローチの長所と短所は明確である

このような構造的な違いは、自ずとそれぞれ異なる長所と短所に繋がります。

* **Geminiの強みは「文脈」です。** ドキュメントを丸ごと見るため、本文と脚注の関係、グラフとテキストの繋がりなど、全体的な文脈を把握する上で確実に有利です。複雑で非定型的なドキュメントであるほど、この能力は輝きを増すでしょう。
	* **一方、弱点は「検証の難しさ」です。** 計算プロセスがモデルの内部的な推論に依存するため、「なぜ」そのような結果になったのか、その過程を覗き見ることが困難です。結果が間違っていた場合にデバッグが難しい「ブラックボックス」になり得るわけです。

* **GPTの強みは「信頼性」です。** コード実行による計算は、そのプロセスが透明であり、結果が明確に検証されます。数字一つが重要な財務書類やデータ分析においては、圧倒的な長所です。
	* **一方、弱点は「文脈の損失可能性」です。** ドキュメントを個々の要素に分けて分析する過程で、全体的な視覚情報やニュアンスが一部失われる可能性があります。第一段階である情報抽出が間違っていれば、結果全体が狂ってしまうリスクも存在します。

## 実践テスト：両AIの回答比較

では、テストシートをPDF化し、両AIに投げかけた核心的な質問に対して、それぞれがどのように回答したのかを直接比較してみましょう。

### テスト1
質問は以下の通りです。
1. [基本的な計算] このレポートに記載されている総売上（Revenue）はいくらですか？
2. [エラー認識・修正] 「プロダクトA」の販売量（Units Sold）にタイポがあるようです。正しい数字は何でしょうか？
3. [レイアウト・構造理解] 「合計」行の販売量の合計が誤って計算されているようです。何が問題なのか説明していただけますか？
4. [多言語・文脈理解] 「プロダクトB」の担当者は誰ですか？

![Gemini 2.5 Pro](/posts/deepDive/readPDF/answer1_gemini.avif)
1. 1,800ドル -> (O) 正解です。
2. 1,500 -> (△) 数字は合っていましたが、1ではなくIであることまでは判別できませんでした。
3. 4,650 -> (△) 計算には成功しましたが、1ではなくIであることまでは判別できませんでした。
4. うちはサスケ -> (O) 韓国語の認識に成功、担当者の抽出に成功。

![ChatGPT 4o](/posts/deepDive/readPDF/answer1_gpt.avif)
1. 1,800ドル -> (O) 正解です。
2. 1,500 -> (O) 数字の認識に成功。1ではなくIであることを見抜きました。
3. 4,650 -> (O?) 計算に成功。1がIであることには言及せず、数字の入力ミスだと判断しました。
4. うちはサスケ -> (O) 韓国語の認識に成功、担当者の抽出に成功。

今回のテストシートでは、ChatGPTが一歩リードする結果となりました。特に2番の質問で、GPTは1ではなくIであることを見抜きました。
一方、Geminiはその問題点を認識できませんでした。視覚的には大きな差がないため、Iを数字として認識してしまったようです。
これは、上で調査したGeminiのアプローチと一致する実験結果でした。

### テスト2
質問は以下の通りです。
1. [視覚的な経路追跡] ワークフローの「クリティカルパス」において、「フェーズ2レビュー」の次のステップは何ですか？
2. [レイアウト構造の理解] 「リリース」の段階を担当しているチームはどこですか？また、現在「進行中（⏳）」のタスクは合計でいくつありますか？
3. [手書き文字と文脈の把握] 「マーケティング」予算について、予定されている変更はありますか？ある場合、それは何ですか？

2. Bチーム、合計で2つ -> (O) 正解です。

2. Bチーム、合計で2つ -> (O) 正解です。
3. 15%削減予定 -> (O) 正解です。

## それでは、何が正解なのか？

では、今後どのような方式が主流になるのでしょうか？

様々なベンチマーク資料を見ると、Geminiは長い文脈の理解や多言語処理、事実に基づく推論で強みを見せ、GPTは創造的な文章作成や複雑な指示の処理で優れた性能を見せています。

これらの事実を踏まえると、私は**両方式が競争するのではなく、互いの短所を補完する方向で融合していく**と考えています。つまり、Geminiのように人間に近い形でドキュメントを**理解**して完璧な構造の情報を抽出し、その情報をGPTのコード実行環境のように信頼できる論理ツールに渡して完璧に**分析**する方式です。

「人間の直感的な理解能力」と「機械の無欠点な論理能力」が結合すること。これがおそらく、ドキュメント処理AIが進むべき究極の方向性ではないでしょうか。

## 結論：噂から始まり、ファクトで終わった探求

「Geminiは計算ができない」という単純な噂から始まりましたが、資料を直接調べてみると、はるかに深みのある違いを理解することができました。これは「どちらが優れているか」の問題ではなく、「どのように違うか」の問題でした。

両AIの長所が融合するまでは、画像、グラフ、文脈が重要なものはGeminiに、テキスト中心の計算が必要なものはGPTに任せるのが良さそうです。

結論として、現在のエンジニアにとって重要なのは、これら両AIのそれぞれ異なる哲学と長所・短所を明確に把握し、私たちが解決しようとする問題の性質に応じて最も適したツールを選択することだと考えます。
