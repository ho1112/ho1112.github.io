2. **[Animating Zooming using CSS: Transform Order is Important… Sometimes](https://frontendfoc.us/link/170531/web)** (출처: *Frontend Focus #697*)

    ![고양이 이미지를 활용한 CSS Transform 애니메이션 예시](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/v1750240479/m11wnghg8nmbk8enxczd.jpg)

    * **배경**: CSS `transform`을 사용해 확대/축소 애니메이션을 구현할 때, 속성의 순서가 예기치 않은 결과를 낳을 수 있습니다. 이는 브라우저가 애니메이션을 처리하는 복잡한 알고리즘 때문입니다.
    * **핵심 기능**: Jake Archibald는 `scale`과 `translate`를 함께 사용할 때 발생하는 문제를 분석하고, `transform` 순서가 애니메이션의 자연스러움에 미치는 영향을 설명합니다.
    * **코드 예시**:
        ```css
        /* 잘못된 순서 - 부자연스러운 움직임 유발 가능 */
        .animate {
          transition: transform 1s;
          transform: scale(2) translate(50px, 50px);
        }

        /* 올바른 순서 - 예측 가능한 자연스러운 움직임 */
        .animate:hover {
          transform: translate(50px, 50px) scale(2);
        }
        ```
    * **실무 적용**: 복잡한 UI 애니메이션을 만들 때 요소가 어색하게 움직이는 문제를 피하고, 사용자가 예측 가능하고 자연스럽다고 느끼는 인터랙션을 구현할 수 있습니다. 특히 줌, 패닝 등 여러 변형이 동시에 일어나는 인터페이스에 유용합니다.
    * **주의사항**: `transform`의 순서를 잘못 지정하면 애니메이션의 중간 단계가 개발자의 의도와 다르게 보일 수 있으므로, 여러 변형을 결합할 때는 항상 순서를 신중하게 고려해야 합니다.

## 2. 🛠️ 툴 & 라이브러리

* **[npmgraph: A Tool to Visualize npm Module Dependencies](https://javascriptweekly.com/link/170446/web)** (출처: *JavaScript Weekly #740*)

    ![npmgraph 툴 사용 예시 화면](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/ndksybgvl78ihuxomdnz.jpg)

    * **개요**: npm 패키지 의존성을 시각적으로 분석할 수 있는 웹 기반 도구입니다. 하나 이상의 패키지 이름이나 `package.json` 파일을 입력하면, 해당 패키지들의 의존성 그래프와 그들이 어떻게 교차하는지를 한눈에 볼 수 있습니다.
    * **사용법**:
        1.  [npmgraph.js.org](https://javascriptweekly.com/link/170446/web)에 접속합니다.
        2.  분석하고 싶은 패키지 이름을 입력하거나 `package.json` 파일을 업로드합니다.
        3.  생성된 의존성 그래프를 탐색합니다.
    * **실전 팁**: 프로젝트의 의존성 구조를 파악하거나, 특정 패키지가 왜 설치되었는지 추적할 때 매우 유용합니다. 유지보수자 수와 같은 기준으로 패키지에 색상을 입히거나, 그래프를 SVG로 다운로드하여 문서화에 활용할 수도 있습니다.

* **[Critical CSS Generator](https://frontendfoc.us/link/170550/web)** (출처: *Frontend Focus #697*)
    * **개요**: 웹 페이지의 초기 렌더링 속도를 향상시키기 위해 'Critical CSS'(사용자가 처음 보는 화면에 필요한 최소한의 CSS)를 추출해주는 온라인 도구입니다.
    * **사용법**:
        1.  분석할 페이지의 URL을 입력합니다.
        2.  뷰포트의 너비와 높이를 지정합니다.
        3.  CSS를 추출하기 전 대기 시간(Render Wait Time)을 설정합니다.
        4.  'Generate Critical CSS' 버튼을 클릭하면 최적화된 CSS가 생성됩니다.
    * **실전 팁**: 생성된 Critical CSS를 HTML 문서의 `<head>` 안에 `<style>` 태그로 직접 삽입하면, 외부 CSS 파일 로드를 기다리지 않고도 페이지의 핵심 콘텐츠를 빠르게 스타일링할 수 있어 '첫 콘텐츠 페인트(FCP)' 시간을 크게 단축시킬 수 있습니다.
