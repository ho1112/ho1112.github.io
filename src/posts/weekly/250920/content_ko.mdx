## 🔥 이번 주 핵심 테마
이번 주는 npm 생태계를 위협하는 공급망 공격이 계속해서 확산되며 보안에 대한 경각심을 높였고, Safari 26.0의 새로운 기능들이 대거 공개되어 웹 개발의 미래를 엿볼 수 있었습니다. 더불어 UX 리서치의 영향력을 측정하고 개선하는 방법에 대한 깊이 있는 논의도 활발했습니다.

## 1. 🔍 심층 분석
- **[The 'Shai-Hulud' npm Supply Chain Attack Rumbles On](https://javascriptweekly.com/link/174584/web)** (출처: *JavaScript Weekly #753*)
    - **배경**: npm 생태계에 영향을 미치는 악의적인 공급망 공격이 계속해서 규모를 키우고 있습니다. '듄'의 모래벌레 이름을 딴 이 공격은 개발자의 컴퓨터에서 토큰과 비밀 정보를 빼내기 위해 수백 개의 패키지에 영향을 미쳤습니다.
    - **핵심 기능**: 공격은 주로 개발 환경의 환경 변수, 특히 `AWS_ACCESS_KEY_ID`, `OPENAI_API_KEY`, `NPM_TOKEN` 등 민감한 정보를 탈취하는 데 초점을 맞추고 있습니다. 이는 CI/CD 환경뿐만 아니라 로컬 개발 환경까지 위협합니다.
    - **실무 적용**: pnpm과 같은 패키지 매니저는 `minimumReleaseAge`와 같은 설정을 도입하여 아주 최근에 배포된 패키지의 설치를 지연시키는 방식으로 대응하고 있습니다. 개발팀은 의존성 관리에 더욱 신중을 기하고, 의심스러운 패키지 사용을 피해야 합니다.
    - **주의사항**: 이 공격은 계속 진화하고 있으므로, npm 패키지를 설치하거나 업데이트할 때 각별한 주의가 필요합니다. 의존성 버전을 고정하고, 정기적인 보안 감사를 수행하는 것이 중요합니다.

- **[WebKit Features in Safari 26.0](https://frontendfoc.us/link/174372/web)** (출처: *Frontend Focus #709*)
    ![WebKit Features in Safari 26.0](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/v1758101976/cfjcdqmjupsyuw2rwe8i.png)
    - **배경**: macOS 26, iOS 26과 함께 Safari 26.0이 출시되면서 웹킷에 다양한 새 기능이 추가되었습니다. 이번 업데이트는 웹 앱, CSS, 3D 모델링 등 여러 분야에 걸쳐 중요한 발전을 포함합니다.
    - **핵심 기능**: 주요 기능으로는 CSS 앵커 포지셔닝, 스크롤 기반 애니메이션, 3D 모델을 웹 페이지에 삽입할 수 있는 `<model>` 요소, CSS `progress()` 함수, SVG 아이콘 지원 등이 있습니다. 또한, 이제 iOS와 iPadOS에서 모든 웹사이트를 홈 화면에 추가하여 웹 앱처럼 사용할 수 있게 되었습니다.
    - **실무 적용**: 앵커 포지셔닝을 사용하면 툴팁이나 팝오버를 특정 요소에 쉽게 고정할 수 있으며, `<model>` 요소를 활용해 인터랙티브한 3D 경험을 제공할 수 있습니다. 웹 앱 기능 강화는 네이티브 앱과 웹 앱의 경계를 더욱 허물 것입니다.
    - **주의사항**: 새로운 기능들은 대부분 최신 버전의 Safari에서만 지원되므로, 프로덕션 환경에 적용하기 전에 브라우저 호환성을 반드시 확인해야 합니다.

## 3. 📝 추천 아티클 & 팁
- **[What Can We Actually Do With `corner-shape`?](https://frontendfoc.us/link/174374/web)** (출처: *Frontend Focus #709*)
    + **핵심 인사이트**: `border-radius`를 넘어 요소의 모서리를 다양하게 스타일링할 수 있는 `corner-shape` CSS 속성을 소개합니다. 현재는 Chrome/Edge에서만 지원되지만, 이 속성을 사용하면 `angle`, `scoop` 등 다양한 형태로 모서리를 꾸밀 수 있어 창의적인 UI 디자인이 가능해집니다. 이 글은 여러 데모를 통해 `corner-shape`의 잠재력을 보여주며, 예상보다 훨씬 더 많은 활용 가능성이 있음을 증명합니다.
    + **왜 이 글이 유익한가**: 표준으로 자리 잡기 전인 새로운 CSS 기능을 미리 학습하고, 미래의 웹 디자인 트렌드를 준비할 수 있는 좋은 기회를 제공합니다.

- **[Follow-the-Leader Pattern with CSS Anchor Positioning](https://frontendfoc.us/link/174381/web)** (출처: *Frontend Focus #709*)
    ![Follow-the-Leader Pattern Demo](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/v1758113235/bdmfvohdb5v6zkzbtui9.jpg)
    + **핵심 인사이트**: CSS 앵커 포지셔닝을 사용하여 '따라다니는' 요소를 만드는 'Follow-the-Leader' 패턴을 소개합니다. 이 패턴은 사용자의 입력이나 상태 변화에 따라 한 요소(follower)가 다른 요소(leader)를 동적으로 따라다니도록 하는 기술입니다. JavaScript 없이 순수 CSS만으로 복잡한 UI 상호작용을 구현할 수 있는 앵커 포지셔닝의 강력함을 보여줍니다. (현재 Chromium 계열 브라우저에서만 지원)
    + **왜 이 글이 유익한가**: 복잡한 DOM 구조나 JavaScript 계산 없이도 유연한 UI를 구축할 수 있는 현대적인 CSS 기법을 배울 수 있습니다.

- **[Affinity Mapping In UX](https://medium.com/design-bridges/affinity-mapping-b0b9faddccfb)** (출처: *Smashing Newsletter #525*)
    ![Affinity Mapping Example](https://mcusercontent.com/16b832d9ad4b28edf261f34df/images/784395d4-f79c-2605-9405-b0b2452c1abf.png)
    + **핵심 인사이트**: 고객 경험 리서치나 브레인스토밍에서 얻은 통찰력을 정리하는 강력한 도구인 어피니티 매핑(친화도 매핑)을 제대로 활용하는 방법을 설명합니다. 단순히 메모를 주제별로 그룹화하는 것을 넘어, 양질의 노트를 작성하고, 캔버스에 체계적으로 정리하며, 클러스터를 통해 계층 구조를 만드는 7단계 접근법을 제시합니다.
    + **왜 이 글이 유익한가**: 리서치 결과를 실행 가능한 결정으로 이끌어내는 체계적인 방법을 배울 수 있어, 팀의 의사결정 과정을 개선하는 데 실질적인 도움이 됩니다.
