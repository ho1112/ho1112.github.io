1. **[A Guide to Scroll-Driven Animations with Just CSS](https://frontendfoc.us/link/170836/web)** (출처: *Frontend Focus #698*)
   ![스크롤 애니메이션 가이드 이미지](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/v1750845956/sgvcpphkcnqmascvhqtk.png)
    - **배경**: 사용자의 스크롤 상호작용에 따라 동적인 애니메이션을 구현하기 위해 JavaScript에 의존하던 기존 방식과 달리, CSS만으로 이를 구현할 수 있는 새로운 웹 표준 기능이 등장했습니다.
    - **핵심 기능**: `scroll()`과 `view()`라는 두 가지 새로운 CSS 함수를 사용하여 스크롤 위치에 따라 요소의 스타일(예: 투명도, 위치)을 변경할 수 있습니다. 이를 통해 스크롤 진행률 표시줄, 뷰포트 진입 시 요소 애니메이션 등을 순수 CSS로 구현 가능합니다.
    - **실무 적용**: 랜딩 페이지의 인터랙티브한 소개 섹션이나, 사용자가 긴 콘텐츠를 읽을 때 상단에 독서 진행률을 시각적으로 보여주는 UI를 만드는 데 유용합니다.
    - **주의사항**: 아직 모든 브라우저에서 완벽하게 지원되지 않을 수 있으며, 과도한 움직임은 사용자의 멀미를 유발할 수 있으므로 접근성을 고려하여 `prefers-reduced-motion` 미디어 쿼리를 함께 사용하는 것이 권장됩니다.

## 2. 🛠️ 툴 & 라이브러리
1. **[Biome v2: The First Type-Aware Linter That Doesn't Require `tsc`](https://javascriptweekly.com/link/170735/web)** (출처: *JavaScript Weekly #741*)
    - **개요**: Biome v2는 타입스크립트 컴파일러(`tsc`)를 필요로 하지 않으면서도 타입 인식 린팅 규칙을 제공하는 최초의 JavaScript 및 TypeScript 린터입니다. 이를 통해 더 빠른 린팅 성능을 기대할 수 있습니다.
    - **주요 특징**:
        - `tsc` 의존성 제거로 인한 성능 향상
        - 린터 플러그인 초기 지원
        - 모노레포 지원 개선
    - **참고**: 현재 Vue 및 Svelte 템플릿은 아직 지원되지 않으므로 해당 프레임워크 사용자들은 주의가 필요합니다.

2. **[\<syntax-highlight\>: A Custom Element for Syntax Highlighting](https://frontendfoc.us/link/170858/web)** (출처: *Frontend Focus #698*)
    - **개요**: 기존의 구문 강조 라이브러리들이 모든 토큰을 `<span>` 태그로 감싸는 방식과 달리, 이 커스텀 엘리먼트는 최신 브라우저에서 지원하는 CSS Custom Highlight API를 사용하여 더 효율적이고 시맨틱한 구문 강조를 제공합니다.
    - **사용법**:
      ```html
      <syntax-highlight language="js">
        const greet = () => "Hello, World!";
      </syntax-highlight>
      ```
    - **실전 팁**: 대부분의 모던 브라우저에서 지원되므로, 블로그나 기술 문서 사이트에서 코드 스니펫을 보여줄 때 성능 저하 없이 깔끔하게 구문 강조를 적용하는 데 유용합니다.

## 3. 📝 추천 아티클 & 팁
1. **[Lightly Poking at the CSS `if()` Function in Chrome 137](https://frontendfoc.us/link/170846/web)** (출처: *Frontend Focus #698*)
    - **핵심 인사이트**: 최근 Chrome 137에 CSS `if()` 함수가 추가되었습니다. 이 함수를 사용하면 미디어 쿼리 밖에서도 조건부 로직을 CSS 속성 값 내에서 직접 처리할 수 있게 되어, 반응형 디자인이나 테마 변경 시 더 간결하고 강력한 스타일링이 가능해집니다.
    - **코드 예시**:
      ```css
      .element {
        /* 컨테이너 너비가 400px보다 크면 1.5rem, 아니면 1rem */
        font-size: if(container-query(width > 400px), 1.5rem, 1rem);
      }
      ```
    - **왜 이 글이 유익한가**: 아직 실험적인 기능이지만, CSS의 미래를 엿볼 수 있는 강력한 기능의 기본 구문과 사용 가능성을 미리 파악할 수 있습니다.

2. **[How TypeScript Solved Its Global `Iterator` Name Clash](https://javascriptweekly.com/link/170753/web)** (출처: *JavaScript Weekly #741*)
    - **핵심 인사이트**: ES2025에서는 이터레이터 헬퍼 메서드를 포함하는 `Iterator` 클래스가 추가되었습니다. 하지만 이는 TypeScript에 이미 존재하던 `Iterator` 타입과 충돌하는 문제였습니다. 이 글은 TypeScript 팀이 이 이름 충돌 문제를 어떻게 창의적으로 해결했는지 그 과정을 설명합니다.
    - **왜 이 글이 유익한가**: 언어 표준의 발전과 타입 시스템 간의 호환성 문제를 해결하는 과정을 통해, 복잡한 기술적 의사결정에 대한 깊은 이해를 얻을 수 있습니다.
