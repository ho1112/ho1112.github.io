## 1. 🔍 심층 분석
1. **[CSS Cascade Layers Vs. BEM Vs. Utility Classes: Specificity Control](https://www.smashingmagazine.com/2025/06/css-cascade-layers-bem-utility-classes-specificity-control/?utm_source=CSS-Weekly&utm_campaign=Issue-614&utm_medium=web)** (출처: *CSS Weekly #614*)
   ![Cascade Layers](https://css-weekly.com/wp-content/uploads/2025/05/css-cascade-layers-bem-utility-classes-specificity-control.jpg)
    - **배경**: CSS 명시도(Specificity) 관리는 대규모 프로젝트에서 항상 어려운 과제였습니다. BEM, 유틸리티 클래스 같은 방법론이 이를 해결하려 노력해왔지만, 캐스케이드 레이어의 등장은 명시도 제어에 대한 근본적인 해결책을 제시합니다.
    - **핵심 기능**: 캐스케이드 레이어는 `@layer` 규칙을 사용하여 스타일시트를 여러 계층으로 나눕니다. 이 계층의 순서에 따라 스타일 우선순위가 결정되므로, 선택자의 명시도나 소스 순서에 덜 의존하게 됩니다. 이를 통해 서드파티 라이브러리, 테마, 유틸리티, 컴포넌트 스타일을 체계적으로 관리하고 예측 가능하게 만들 수 있습니다.
    - **코드 예제**:
      ```css
      @layer reset, base, components, utilities;

      @layer base {
        p { max-width: 65ch; }
      }

      @layer components {
        .card { background-color: grey; }
      }
      ```
    - **실무 적용**: 디자인 시스템이나 대규모 애플리케이션에서 스타일 충돌을 방지하고 CSS 아키텍처를 단순화하는 데 매우 유용합니다. BEM과 유틸리티 클래스를 레이어 안에서 함께 사용하여 각 방법론의 장점을 취할 수 있습니다.
    - **주의사항**: 레이어 순서가 명시도에 큰 영향을 미치므로, 프로젝트 초기에 전체적인 레이어 구조를 신중하게 설계하는 것이 중요합니다.

2. **[CSS Conditionals with the New `if()` Function](https://frontendfoc.us/link/171454/web)** (출처: *Frontend Focus #700*)
   ![CSS if() Function](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/v1752063792/yhuyagm43dcjnbae2ti1.png)
    - **배경**: CSS에 `if()` 함수가 도입되면서 미디어 쿼리, 컨테이너 쿼리, `supports()` 규칙과 함께 인라인 조건문을 사용할 수 있게 되어 CSS 아키텍처에 새로운 가능성이 열렸습니다.
    - **핵심 기능**: `if()` 함수를 사용하면 특정 조건에 따라 CSS 속성 값을 동적으로 변경할 수 있습니다. 예를 들어, 뷰포트 너비에 따라 다른 색상 값을 적용하는 것을 단 한 줄로 간결하게 처리할 수 있습니다.
    - **코드 예제**:
      ```css
      .element {
        --is-large: if(container(width > 40em), 1, 0);
        padding: calc(var(--is-large) * 2rem + 1rem);
      }
      ```
    - **실무 적용**: 반응형 디자인에서 여러 미디어 쿼리 블록을 사용하거나 복잡한 `calc()` 함수를 사용하는 대신 코드를 더 간결하고 직관적으로 만들 수 있습니다.
    - **주의사항**: Chrome 137 이상에서 지원되므로, 프로그레시브 인핸스먼트 전략과 함께 사용해야 합니다. 모든 브라우저에서 지원되기까지는 시간이 걸릴 수 있습니다.

## 2. 🛠️ 툴 & 라이브러리
- **[Milkdown: A Plugin-Driven WYSIWYG Markdown Editor Framework](https://javascriptweekly.com/link/171374/web)** (출처: *JavaScript Weekly #743*)
  ![Milkdown Editor](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/tubxfmx8h4pqzyydckgu.jpg)
  + **개요**: 플러그인 시스템을 기반으로 하는 WYSIWYG 마크다운 에디터 프레임워크입니다. 높은 수준의 커스터마이징이 가능하며, 공식 문서 자체가 Milkdown으로 렌더링될 정도로 강력합니다.
  + **사용법**: React, Vue, Svelte 등 다양한 프레임워크와 함께 사용할 수 있으며, 핵심 로직은 프레임워크에 독립적입니다. 플러그인을 추가하여 기능을 쉽게 확장할 수 있습니다.
    ```javascript
    import { Editor, rootCtx, defaultValueCtx } from '@milkdown/core';
    import { commonmark } from '@milkdown/preset-commonmark';

    Editor.make()
      .config((ctx) => {
        ctx.set(rootCtx, document.querySelector('#editor'));
        ctx.set(defaultValueCtx, '# Hello world');
      })
      .use(commonmark)
      .create();
    ```
  + **실전 팁**: 실시간 협업, 테이블 편집, 수학 공식 입력 등 다양한 고급 기능을 플러그인을 통해 구현할 수 있어 노트 앱, 문서 도구, 블로그 에디터 등에 적합합니다.

- **[BlockNote 0.33: A 'Notion-Like' Block-Based Text Editor](https://react.statuscode.com/link/171510/web)** (출처: *React Status #435*)
  ![BlockNote Editor](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/qp5pgbk1eisygy6dd3sn.jpg)
  + **개요**: ProseMirror와 TipTap을 기반으로 한 '노션 스타일'의 블록 기반 텍스트 편집기입니다. 드래그 앤 드롭, 실시간 협업, AI 기능, 커스텀 '슬래시 커맨드' 메뉴 등을 제공하여 강력한 편집 환경을 구축할 수 있습니다.
  + **사용법**: React와 함께 사용하기 쉽도록 설계되었습니다. `useBlockNote` 훅을 사용하여 에디터 인스턴스를 만들고 `BlockNoteView` 컴포넌트로 렌더링하면 됩니다.
    ```javascript
    import { BlockNoteView, useBlockNote } from "@blocknote/react";
    import "@blocknote/react/style.css";

    export default function App() {
      const editor = useBlockNote();
      return <BlockNoteView editor={editor} />;
    }
    ```
  + **실전 팁**: 실시간 협업 문서 도구, 블로그 에디터, 노트 앱 등 다양한 애플리케이션에 쉽게 임베드하여 사용할 수 있으며, 플러그인을 통해 기능 확장이 용이합니다.
