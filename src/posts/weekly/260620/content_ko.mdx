## 1. 🔍 심층 분석
- **[Flow for TypeScript Users in 2026](https://javascriptweekly.com/link/186567/web)** (출처: *JavaScript Weekly #790*)
  ![Flow for TypeScript Users](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/nylqu7hc7vyxog0czncx.jpg)
   - 배경: Meta의 타입 시스템인 Flow가 수년에 걸쳐 TypeScript와 문법적으로 매우 유사해졌습니다. 본 아티클은 두 시스템의 현재 차이점과 Flow가 가진 고유한 강점을 비교 분석하기 위해 작성되었습니다.
   - 핵심 기능: TypeScript의 `strict` 모드에서도 통과될 수 있는 런타임 충돌 유발 패턴을 기본적으로 차단하며, 누락 없는 분기 처리를 보장하는 완전한 `match` 표현식 같은 독자적 기능을 지원합니다.
   - 코드 예제:
     ```typescript
     // Flow의 패턴 매칭을 통한 철저한 타입 검증 예시
     const result = match (value) {
       case 'A': return 1;
       case 'B': return 2;
     };
     ```
   - 실무 적용: 극도의 타입 안정성과 무결성이 요구되는 대규모 엔터프라이즈 코드베이스를 설계할 때 아키텍처적 영감을 제공합니다.
   - 주의사항: TypeScript의 생태계가 압도적으로 크기 때문에 전면 도입보다는 설계 원칙을 학습하고 적용하는 용도에 적합할 수 있습니다.

- **[Alternatives To Nested Modals](https://uxplanet.org/removing-nested-modals-from-digital-products-6762351cf6de)** (출처: *Smashing Newsletter #563*)
  ![Removing Nested Modals](https://mcusercontent.com/16b832d9ad4b28edf261f34df/images/fae55fef-bb7c-9f16-8fc4-38c3ca99844d.png)
   - 배경: 데이터가 방대하고 복잡한 프로덕트에서는 모달 창 내부에 또 모달 창을 띄우는 중첩 모달(Nested Modals)이 흔히 쓰이지만, 이는 사용자 경험과 맥락을 크게 훼손합니다.
   - 핵심 기능: 중첩 모달이 해결하고자 했던 근본 원인을 해체하고, 사용자가 끊김 없이 탐색할 수 있도록 돕는 사이드 패널이나 인라인 확장형 UI 등 대체 패턴을 소개합니다.
   - 코드 예제:
     ```html
     <aside class="side-panel-overlay">
       <h2>상세 설정 변경</h2>
       </aside>
     ```
   - 실무 적용: 수백 개의 필터나 복잡한 권한 설정을 제공해야 하는 B2B SaaS 앱이나 대시보드의 사용성을 개선할 때 유용합니다.
   - 주의사항: 사이드 패널 방식을 채택할 경우 모바일이나 좁은 해상도에서는 화면 공간 제약이 발생할 수 있으므로 치밀한 반응형 설계가 동반되어야 합니다.

## 2. 🛠️ 툴 & 라이브러리
- **[Introducing the MDN MCP Server](https://developer.mozilla.org/en-US/blog/introducing-mdn-mcp-server/)** (출처: *Frontend Focus #746*)
  ![MDN MCP Server](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/uqqo8fwjphal6xmofqme.jpg)
  + 개요: 대규모 언어 모델이 가진 낡은 훈련 데이터의 한계를 극복하기 위해, MDN 공식 웹 문서와 브라우저 호환성 데이터를 AI 에이전트(Claude 등) 및 IDE에 실시간으로 직접 연동해 주는 서버입니다.
  + 사용법:
    ```json
    // MCP 클라이언트 설정에서 MDN 서버를 추가하여 즉시 사용
    {
      "mcpServers": {
        "mdn": {
          "command": "npx",
          "args": ["@mdn/mcp-server"]
        }
      }
    }
    ```
  + 실전 팁: IDE 내부에서 AI 어시스턴트를 통해 프론트엔드 코드를 작성할 때, 부정확한 코드를 제안받는 대신 MDN의 최신 스펙을 기반으로 한 정확한 코드 및 호환성 피드백을 받을 수 있습니다.

- **[Prop For That: What JS Knows, Now CSS Knows](https://nerdy.dev/prop-for-that)** (출처: *Frontend Focus #746*)
  ![Prop For That](https://res.cloudinary.com/cpress/image/upload/w_1280,e_sharpen:60,q_auto/aqyrlmavjaaiz2h90xtp.jpg)
  + 개요: 마우스 위치, 시간 흐름, 슬라이더 값 등 기존에는 JavaScript를 통해서만 제어할 수 있었던 동적 데이터를 단순한 HTML 속성 부여만으로 CSS Custom Properties(변수)와 연동해 주는 브릿지 라이브러리입니다.
  + 사용법:
    ```html
    <div prop-for-that="mouse-position">
      <p>CSS에서 --mouse-x, --mouse-y 변수로 접근 가능합니다.</p>
    </div>
    ```
  + 실전 팁: JS 이벤트 리스너의 성능 오버헤드를 줄이면서, 순수 CSS만으로 마우스 추적 조명 효과(Glow Effect) 같은 고도화된 UI 인터랙션을 매우 선언적이고 간결하게 구현할 수 있습니다.

## ✍️ 마치며
이번 주 기술 동향을 살펴보면 화려하게 부상하는 신기술의 이면에, '성숙도와 유지보수성'을 고민하는 개발자들의 고뇌가 짙게 깔려 있음을 느낄 수 있습니다. React Router v8이 증명하듯 때로는 '지루하고 안정적인' 기술이 비즈니스 환경에서 가장 큰 경쟁력이 될 수 있습니다. 반면 MDN MCP 서버나 Zod 컴파일러의 등장은 도구의 고도화가 개발자 경험(DX)을 얼마나 비약적으로 상승시킬 수 있는지 여실히 보여줍니다. 코드 생성의 비용은 줄어들었지만 아키텍처의 복잡성을 통제해야 하는 책임은 더욱 커진 지금, 유행을 맹신하기보다 팀의 도메인 특성에 맞추어 검증된 낡은 기술และ 파괴적인 신기술 사이에서 현명하게 줄타기를 하는 통찰력이 그 어느 때보다 요구되는 시점입니다.
