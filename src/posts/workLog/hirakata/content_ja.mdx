<h1>히라가나↔카타카나로 변환</h1>

기존에는 유저명을 한자(+히라가나) 조합으로 표시하고 있었는데, 클라이언트쪽에서 한자 표기만으로는 읽는 방법을 알 수 없으니 후리가나 표기를 추가해달라는 문의가 있었다.
마케팅부에서 이 안건을 진행하면서 타 시스템와의 연계・통합을 위해 카타카나로 표기해달라는 요청이 왔다.

하지만 DB에는 히라가나 데이터 밖에 갖고 있지 않았기 때문에 이것을 카타카나로 변환하는 처리를 해주어야 했다.

그래서 과제는 크게 2가지
- 서버에서 받은 히라가나 데이터를 카타카나로 변환
- 기존 유저명 표기 위해 후리가나 표기를 추가

---
## 히라가나와 카타카나
우선 히라가나를 판정해주기 위해 유니코드를 살펴보았다.
![히라가나](/posts/workLog/hirakata/hiragana.avif)
![카타카나](/posts/workLog/hirakata/katakana.avif)
히라가나와 카타카나의 유니코드 값의 차이는 `0x60`임을 알 수 있다.
(블록당 16개, 16*6이므로 십진수로는 96차이)

즉, 히라가나 'あ'(U+3042)를 카타카나 'ア'(U+30A2)로 변환할 때는, 'あ'의 유니코드 값에 0x60을 더해주면 된다.
U+3042 (히라가나 'あ') + 0x60 = U+30A2 (카타카나 'ア')

[히라가나 유니코드 표](https://en.wikipedia.org/wiki/Hiragana_(Unicode_block))
[카타카나 유니코드 표](https://en.wikipedia.org/wiki/Katakana_(Unicode_block))

---
### 히리가나를 카타카나로 변환
히라가나의 유효 대상범위는 3041~3096로 지정해주었다.
1. 히라가나인지 체크
2. `0x60`을 더해서 리턴
```javascript
// ひらがなをカタカナに変換
export const convertToKatakana = (input: string): string => {
  return input.replace(/[\u3041-\u3096]/g, (match) =>
    String.fromCharCode(match.charCodeAt(0) + 0x60)
  )
}
```
### 카타카나를 히라가나로 변환
이번 안건의 대상외지만 카타카나를 히라가나도 변환할 때는 반대로 `-0x60`를 해주면 된다.
카타카나의 유효 대상범위는 30A1~30F6로 지정
```javascript
// カタカナをひらがなに変換
export const convertToHiragana = (input: string): string => {
  return input.replace(/[\u30A1-\u30F6]/g, (match) =>
    String.fromCharCode(match.charCodeAt(0) - 0x60)
  )
}
```
### 히라가나↔카타카나
두 가지를 동시에 사용하는 시스템은 거의 없을 것 같지만
이렇게 해버리면 공통함수로 만들어 버릴 수도 있다.
```javascript
// ひらがなとカタカナを相互に変換
export const toggleKana = (input: string): string => {
  return input.replace(/[\u3041-\u3096\u30A1-\u30F6]/g, (match) => {
    const code = match.charCodeAt(0);
    return String.fromCharCode(code + (code < 0x3097 ? 0x60 : -0x60));
  });
}
```

### 후리가나 표기(+루비태그)
이렇게 변환해준 카타카나를 유저명 위에 표기를 해주어야한다.
후리가나를 표기할 때 대표적으로 사용되는 것으로는 `<ruby>`[루비태그](https://developer.mozilla.org/ko/docs/Web/HTML/Element/ruby)가 있다.

```html
<ruby>平仮名<rp>(</rp><rt>ひらがな</rt><rp>)</rp></ruby>
```
<ruby>平仮名<rp>(</rp><rt>ひらがな</rt><rp>)</rp></ruby>
이런 방식으로 `<ruby>`, `<rt>`, `<rp>`의 조합으로 후리가나를 표기해줄 수 있다.

그런데 이 루비태그를 그대로 사용하기에는 문제점이 있었다.
1. 디자인 커스터마이징에 한계가 있다.
2. 브라우저간의 차이가 발생할 수 있다.
3. 해당 한자와 매칭하지 않고 왼쪽으로 정렬해서 표기하고 싶다.
4. 유저 정보 상세페이지에 표시할 공간이 적기 때문에 이름이 긴 유저(외국인 유저 등)의 경우 2줄 표기로 한다.

-> 그래서 결과적으로는 루비태그를 채용하지 않고 직접 만들어서 표기하는 방법을 사용했다.
기존의 유저명 위에 태그 하나 추가해준 뒤 CSS만으로 제어했기 때문에 `<rt>`, `<rp>` 사용해야 하는 루비태그에 비해 간결하게 표현이 가능했다.

## おわりに
히라가나와 카타카나 간의 변환은 유니코드의 고정된 오프셋 값을 활용하여 간단하게 처리할 수 있었다.
유니코드의 일정한 수치 차이를 이용하는 방법은 간결하고 신뢰도 또한 높기 때문에 다른 문자처리에도 적용할 수 유연성을 갖춘 방법이라고 생각한다.
